## 工作流 1：竞品系统分析

### 目标
快速了解一款游戏的系统架构

### 步骤
1. **设定研究范围**（5分钟）
   - 选择要分析的游戏
   - 确定要研究的系统（战斗/经济/社交）

2. **AutoGPT 自动收集**（自动运行）
   ```
   研究《游戏名》的XXX系统：
   1. 搜索 5+ 篇深度分析文章
   2. 提取核心机制和数值结构
   3. 分析优缺点
   4. 输出结构化报告
   ```

3. **人工验证**（30分钟）
   - 检查关键事实
   - 补充个人体验
   - 标注可信度

4. **整理入库**（15分钟）
   - 保存到 `03-research/competitor-analysis/`
   - 提取可复用的设计模式到笔记

### 输出模板
```markdown
## 游戏名：XXX

### 系统概述
一句话描述系统核心循环

### 核心机制
| 机制 | 说明 | 数值（如有） |
|------|------|-------------|
| 机制A | ... | ... |

### 优缺点分析
- ✅ 优点：...
- ❌ 缺点：...

### 可借鉴之处
- ...

### 可信度
- 信息来源：...
- 验证状态：已验证/待验证/推测
```

## 工作流 2：概念图生成

### 目标
快速生成游戏角色/场景概念图用于验证

### 步骤
1. **明确需求**
   - 角色/场景/道具？
   - 风格：写实/二次元/像素？
   - 关键特征描述

2. **撰写 Prompt**
   ```
   masterpiece, best quality, concept art,
   [主体描述], [风格], [特征],
   [光照], [背景],
   --ar 16:9 --niji 5
   ```

3. **批量生成**（10-20 张）
   - 使用不同随机种子
   - 微调 Prompt 对比效果

4. **筛选与迭代**
   - 选择 3-5 张方向对的
   - 局部重绘优化
   - ControlNet 控制构图

5. **文档配图**
   - 保存到设计文档
   - 标注生成参数（便于复现）

## 工作流 3：头脑风暴

### 目标
快速产生创意并评估

### 方法
1. **发散阶段**（用 AI）
   ```
   我正在设计一个[类型]游戏，
   想实现[核心体验]。
   请帮我头脑风暴 10 个创新的机制点子。
   ```

2. **收敛阶段**（人工）
   - 筛选可行性高的 3-5 个
   - 快速原型验证
   - 选择 1-2 个深入设计

3. **记录**（用 mem0）
   - 保存所有点子到知识库
   - 标注评估结果
   - 废弃点子也不删除（未来可能有用）
