## 核心设计理念

### 情绪作为UI系统

The Alters 最核心的设计突破是将**情绪表现作为用户界面**:

- **传统UI**: 红黄绿状态图标显示Alter情绪
- **情感UI**: 叹气、表情变化、声音颤抖、停顿等微表情
- **结果**: 玩家几乎不依赖图标，而是通过表演感知情绪状态

> 设计公式: `情感表演反馈 = 视觉UI × 10`

---

## 关键量化数据

### 情绪-工时换算表

| 情绪状态 | 工作时长 | 特点 |
|---------|---------|------|
| Joyful (快乐) | 12小时 | 可承受"加班"(crunch) |
| Content (满足) | 9小时 | 标准产出 |
| Stressed/Anxious (焦虑) | 6小时 | 效率下降 |
| Furious/Depressed (愤怒/抑郁) | 4小时 | 可能叛变或死亡 |

**关键洞察**: 情绪不是装饰性数值，而是直接影响生产力的核心资源。

### 加班系统

- 平均耐受: **2天**高强度工作
- 个体差异: 科学家可在高基线情绪下稳定工作，技术员和植物学家更快不耐烦
- 后果: 长期压力导致叛变或游戏结束

## 系统架构: 身份经济

### 双层经济模型

```
传统生存游戏: 资源经济(时间 + 材料 + 安全)
The Alters:    资源经济 + 身份经济(情感温度管理)
```

**身份经济运作方式**:
1. 情绪状态 → 工作时长 → 资源产出
2. 任务分配 → 情绪消耗 → 需维护成本
3. 社交互动 → 情绪恢复 → 投资回报率

### 反馈循环

```
玩家心理 → 系统决策 → 情感输出 → 玩家理解 → 任务分配 → 系统响应
```

**设计要点**: 系统不硬编码"关系值"，而是通过重复接触让玩家自然形成对不同Alter的认知。

## 时间资源管理

### 工作窗口

| 时间段 | 可用时间 | 限制 |
|-------|---------|------|
| 白天 | 7am-9pm (14h) | 无辐射风险 |
| 太阳靠近后 | 7am-7pm (12h) | 夜间辐射提前开始 |
| 室内任务 | 弹性 | 取决于Alter可用性和情绪 |

### 资源压力窗口

- **有机物**: 每日节奏(需持续关注)
- **金属/矿物/rapidium**: 1-2天缓冲期
- **交错时间线**: 创造轮换瓶颈而非单点阻塞

---

## 情感成本设计

### 任务压力循环

1. **日常压力**: 修理、食物、存储、模块故障
2. **外部压力**: 夜间辐射、异常点、时间流逝
3. **内部压力**: Alter情绪、工作小时减少
4. **复合效应**: 小失误在多个系统间放大

### 食品质量影响

- 1-2天"糊"(mush) → 士气下降 → 次日工作时长减少
- 连锁反应: 资源短缺 → 更多外出时间 → 更少的室内维护时间

## 环境叙事设计

### 室外环境

**Rapidium资源设计**:
- 视觉上: 像虚空一样
- 感觉上: 空气更沉重
- 行为上: 玩家会自动加快工作速度离开
- **设计目的**: 无威胁但产生心理压力

**异常点(Anomalies)**:
- 声音设计: 穿透安静
- 行为反应: 让玩家突然转身
- 危险布局: 间隙过窄，突然移动会撞上

### 室内环境

**走廊设计**:
- 狭窄 → 移动紧凑高效
- 细节分布 → 不 rushing 时才能发现
- 电梯动画 → 速度恰到好处，不会想跳过

**社交房间**:
- 功能: 唯一"人性化"空间
- 音乐: 更轻快
- 互动: 啤酒乒乓球、看电影
- **设计意图**: 对比生存压力，创造情感锚点

## 任务管理结构

### 户外任务规划

```
日出 → 规划路线 → 资源收集 → (9pm前)返回基地
         ↓
    时间 = 最宝贵资源
```

### 室内任务分配逻辑

当模块损坏时:
- 问题: 哪个Alter来做修理？
- 决策: "谁现在在做的工作最不关键？"
- 系统逻辑: 动态优先级重排

## 设计启示

### 可复用模板

**情感-系统整合检查清单**:
- [ ] 情绪是否影响核心玩法循环(不只是装饰)？
- [ ] 是否有量化的情绪-产出换算？
- [ ] 玩家是否能通过非UI方式感知系统状态？
- [ ] 是否有情感恢复机制需要玩家主动投资？
- [ ] 环境设计是否创造情感压力/缓解？

**生存游戏分层经济模型**:
```
Layer 1: 基础资源(食物/材料/工具)
Layer 2: 时间资源(白天/季节/事件窗口)
Layer 3: 人力资源(情绪/技能/关系)
Layer 4: 叙事资源(选择/后果/身份)
```

## 行动建议

1. **立项时**: 明确定义核心情感目标(如"存在恐惧"、"内省"、"道德冲突")
2. **系统设计时**: 每个子系统都要问"这如何支持/削弱核心情感？"
3. **数值设计时**: 建立情绪-效率的量化换算表
4. **测试时**: 验证玩家是否能不依赖UI感知系统状态
5. **迭代时**: 检查叙事与系统是否指向同一情感方向
