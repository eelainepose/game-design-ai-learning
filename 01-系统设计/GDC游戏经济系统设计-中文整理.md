## 核心概念：双货币系统

### 什么是双货币系统？

绝大多数成功的免费游戏（F2P）都采用**双货币机制**：

| 货币类型 | 获取方式 | 用途 | 设计目的 |
|---------|---------|------|---------|
| **软货币** (Soft Currency) | 游戏内获得 | 常规消费 | 游戏循环、成就感 |
| **硬货币** (Hard Currency) | 现金购买 | 特殊/加速 | 变现 |

**典型案例**：
- 《部落冲突》：金币（软）+ 宝石（硬）
- 《王者荣耀》：金币（软）+ 点券（硬）
- 《原神》：摩拉（软）+ 原石（硬）

---

## 为什么需要双货币？

### 1. 心理定价效应
- **软货币**：玩家感觉"免费"，愿意大量消费
- **硬货币**：与真实货币挂钩，消费更谨慎

### 2. 经济隔离
- 防止通货膨胀影响付费玩家体验
- 保护付费玩家的投入价值

### 3. 行为引导
- 软货币驱动日常活跃
- 硬货币提供捷径和便利

## 经济系统设计原则

### 1. 产出与消耗平衡

```
经济健康度 = 产出速率 / 消耗速率

理想比例：1.1 - 1.3（10%-30%盈余）
```

**产出来源**：
- 游戏玩法奖励
- 每日任务
- 成就系统
- 活动赠送

**消耗去向**：
- 装备升级
- 角色培养
- 抽卡/开箱
- 复活/加速

### 2. 三种经济循环

#### 核心循环（Core Loop）
```
玩游戏 → 获得软货币 → 升级 → 更强 → 玩更难的内容
```

#### 留存循环（Retention Loop）
```
每日登录 → 领取奖励 → 完成日常 → 积累进度
```

#### 变现循环（Monetization Loop）
```
遇到瓶颈 → 等待/ grind → 或付费加速 → 突破瓶颈
```

### 3. 硬货币投放策略

| 投放方式 | 数量 | 目的 |
|---------|------|------|
| 新手礼包 | 少量 | 教学使用 |
| 成就奖励 | 中等 | 里程碑激励 |
| 活动赠送 | 周期性 | 回流召回 |
| 维护补偿 | 应急 | 玩家安抚 |

**黄金法则**：让玩家习惯使用硬货币，但不要赠送太多。

## 数值设计方法

### 1. 软货币曲线设计

**经验曲线公式**：
```
Level_n = Level_{n-1} × ratio

ratio 选择：
- 缓慢增长：1.1（休闲游戏）
- 中等增长：1.2（RPG）
- 快速增长：1.5（硬核游戏）
```

**示例：RPG 经验需求**
| 等级 | 经验需求 | 累计经验 |
|-----|---------|---------|
| 1 | 100 | 100 |
| 2 | 120 | 220 |
| 3 | 144 | 364 |
| 4 | 173 | 537 |
| 5 | 207 | 744 |

### 2. 硬货币汇率设计

**汇率确定因素**：
- 最低付费档位（如 6 元）
- 首充翻倍优惠
- 大额购买的折扣率

**示例定价**：
| 硬货币数量 | 价格 | 性价比 |
|-----------|------|--------|
| 60 | ¥6 | 基准 |
| 300 | ¥30 | +首充双倍=600 |
| 980 | ¥98 | 省 2% |
| 1980 | ¥198 | 省 5% |
| 6480 | ¥648 | 省 10%+额外赠送 |

### 3. 付费点设计

**冲动消费（Impulse Buy）**：
- 复活：¥6 立即满血
- 加速：跳过等待时间
- 补体力：继续游戏

**计划消费（Planned Buy）**：
- 月卡：每天登录领取
- 战令：完成赛季任务
- 礼包：限时折扣

**大额消费（Whale Buy）**：
- 限定角色/皮肤
- 排行榜竞争
- 公会战优势

## 常见经济问题与解决方案

### 问题 1：通货膨胀
**现象**：软货币贬值，老玩家积累过多

**解决方案**：
- 引入新的消耗渠道
- 赛季重置/新赛季货币
- 限定时间活动消耗

### 问题 2：付费差距过大
**现象**：免费玩家与付费玩家差距悬殊

**解决方案**：
- 软货币可购买大部分内容（只是慢）
- 付费主要节省时间，不是数值碾压
- 技巧/策略可以弥补数值差距

### 问题 3：硬货币囤积
**现象**：玩家攒着不用，等待"最优"消费时机

**解决方案**：
- 限时活动/限定商品
- 硬货币会过期的设计（慎用）
- 制造 FOMO（错失恐惧）

## 经济监控指标

### 关键数据

| 指标 | 健康范围 | 异常信号 |
|-----|---------|---------|
| 日活玩家平均软货币 | 中等 | 过高=通胀，过低=产出不足 |
| 硬货币付费率 | 2-5% | 过低=付费点设计问题 |
| ARPPU（付费用户平均收入） | 因游戏而异 | 持续下降=内容不足 |
| 首充率 | 20-40% | 过低=首充奖励不够吸引 |
| 月卡续费率 | 60%+ | 过低=日常奖励没价值 |

## 实战案例：简化 RPG 经济设计

### 基础设定
- 目标：休闲玩家 30 天达到满级
- 软货币：金币
- 硬货币：钻石

### 数值规划

**升级经验**：
```
总经验需求 = 约 100 万
日均获得 = 100 万 / 30 = 3.3 万
每次战斗 = 500 经验
日均战斗 = 3.3 万 / 500 = 66 场 ≈ 每天 2 小时
```

**付费加速**：
```
跳过等待：10 钻石 = 1 小时
经验药水：100 钻石 = 双倍经验 1 小时
复活：50 钻石
```

**定价**：
```
6 元 = 60 钻石（1 元 = 10 钻）
月卡：30 元 = 300 钻石 + 每天 30 钻石
```

### 预期收入
- 月卡用户：30 元/月
- 轻度付费：100 元/月
- 中重度：500-2000 元/月

## 检查清单

设计经济系统时问自己：
- [ ] 软货币产出是否让玩家有成就感？
- [ ] 硬货币付费点是否合理（不强迫但不难受）？
- [ ] 免费玩家能否体验到完整内容？
- [ ] 付费玩家是否感到物有所值？
- [ ] 经济是否可长期维持（不通胀）？
