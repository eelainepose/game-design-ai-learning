"# 设计模式读书笔记 > 来自 munificent/game-programming-patterns --- ## Command 模式 ### 概念 将请求封装为对象，从而可以： - 参数化客户端 - 队列请求 - 支持撤销操作 ### 游戏应用 - **输入处理**：将按键映射为命令对象 - **回放系统**：记录命令序列实现回放 - **撤销系统**：实现操作撤销 ### 代码示例 ```cpp class Command { public: virtual ~Command() {} virtual void execute() = 0; virtual void undo() = 0; }; ``` --- ## Observer 模式 ### 概念 定义对象间的一对多依赖，当一个对象状态改变时，所有依赖者自动收到通知。 ### 游戏应用 - **成就系统**：监听游戏事件触发成就 - **任务系统**：监听条件变化更新任务状态 - **UI 更新**：数据变化自动刷新界面 --- ## State 模式 ### 概念 允许对象在内部状态改变时改变它的行为。 ### 游戏应用 - **角色状态机**：Idle、Walk、Attack、Hurt - **游戏流程管理**：Menu、Playing、Paused、GameOver - **AI 状态**：Patrol、Chase、Attack --- *待补充更多模式...* "
